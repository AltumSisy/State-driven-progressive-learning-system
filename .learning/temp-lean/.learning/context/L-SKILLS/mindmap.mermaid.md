## 三、深度测试问题（对应README 1.3）

```mermaid
mindmap
    root((**深度测试**))
        **Q1: Skills≠Prompt**
            L0: 放在文件里
            L1: 分层加载
            L2答案:
                脚本能力+工程资产
                把确定性交给程序
                形成可版本化可验收资产
        **Q2: description写太泛**
            L0: 不知道什么时候用

                路由失效→死文件
                必须写清何时用+产出+触发词
        **Q3: 脚本被低估**
            L0: 执行命令
            L1: 减少工作量
            L2答案:
                双指标提升
                脚本不进上下文
                稳定性+成本可控
```

## 四、对抗测试（对应README 3.1-3.2）

```mermaid
mindmap
    root((对抗测试))
        脆弱点诊断
            验收vs确定性混淆
                风险: 中
                补救: 明确验收→输出正确性
            入口文件设计原则
                风险: 中
                补救: 入口短+拆分+单一
            渐进式披露偏差
                状态: 已修正
                补救: 三层加载
        反事实情境
            情境1: 无验收但流程详细
                L2答案: 不算好Skill
                    输出可能缺字段/格式错误
            情境2: 代码审查未触发
                L2答案: description路由失效
                    写太泛没匹配触发词
            情境3: 10任务塞进一个Skill
                L2答案: 入口太长+违反单一职责
                    约束淹没+无法复用
```

## 十、生命周期管理：分级标准（对应README 七.2）

```mermaid
mindmap
    root((Skill分级))
        P0-工程级
            影响: 代码/系统变更
            场景: PR review、发布流程
            验收: 测试命令必须通过
            管理: 两人review
        P1-故障级
            影响: 系统稳定性
            场景: 故障排障、告警处理
            验收: 复盘草稿完整
            管理: 一人review
        P2-进度级
            影响: 团队协作
            场景: 周报、项目状态
            验收: 字段完整即可
            管理: 自检即可
        P3-辅助级
            影响: 效率提升
            场景: 文档整理、格式转换
            验收: 格式符合标准
            管理: 个人维护
```
