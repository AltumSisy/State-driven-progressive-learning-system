# 工具注册中心深度解析

> 基于 `pi-coding-agent` 源码的真实实现，理解内置工具和扩展工具如何汇合到同一个注册中心。

---

## 🎯 核心问题

- 内置工具是如何被注册的？
- 扩展工具是如何被注册的？
- 两条路径如何汇合？
- 工具最终如何被 LLM 调用？

---

## 📐 注册的本质

> **注册中心就是一个 Map 收集器：内置工具和扩展工具都往同一个 Map 里写，最后统一收集。**

```
注册 = 往 Map 里写一条记录

extension.tools.set(toolName, {
  definition: ToolDefinition,
  sourceInfo: SourceInfo
})
```

```typescript
export function createAllToolDefinitions(cwd: string, options?: ToolsOptions): Record<ToolName, ToolDef> {
  return {
    read: createReadToolDefinition(cwd, options?.read),
    bash: createBashToolDefinition(cwd, options?.bash),
    edit: createEditToolDefinition(cwd, options?.edit),
    write: createWriteToolDefinition(cwd, options?.write),
    grep: createGrepToolDefinition(cwd, options?.grep),
    find: createFindToolDefinition(cwd, options?.find),
    ls: createLsToolDefinition(cwd, options?.ls),
  };
}
```

**返回值**：
```typescript
{
  read: ToolDefinition { name, description, execute... },
  bash: ToolDefinition { name, description, execute... },
  ...
}
```

#### ② 存入 Session 内部Map

```typescript
// 文件：agent-session.ts 第2343-2365行 _buildRuntime()

private _buildRuntime(options: {...}): void {
  // 创建所有内置工具定义
  const baseToolDefinitions = createAllToolDefinitions(this._cwd, {
    read: { autoResizeImages },
    bash: { commandPrefix: shellCommandPrefix, shellPath },
  });

  // 存入内部 Map
  this._baseToolDefinitions = new Map(
    Object.entries(baseToolDefinitions).map(([name, tool]) => 
      [name, tool as ToolDefinition]
    ),
  );
  
  // 此时：
  // _baseToolDefinitions = Map {
  //   "read" => ReadToolDefinition,
  //   "bash" => BashToolDefinition,
  //   "edit" => EditToolDefinition,
  //   ...
  // }
}
```

**注意**：此时工具还没有注册到 Extension，只是存在 `_baseToolDefinitions` Map 中！

#### ③ 包装并设置到 Agent

```typescript
// 文件：agent-session.ts 第2252-2341行 _refreshToolRegistry()

private _refreshToolRegistry(): void {
  // 包装内置工具成 AgentTool
  const wrappedBuiltInTools = wrapRegisteredTools(
    Array.from(this._baseToolDefinitions.values())
      .map((definition) => ({
        definition,
        sourceInfo: createSyntheticSourceInfo(`<builtin:${definition.name}>`, 
          { source: "builtin" }
        ),
      })),
    runner,
  );
  
  // 存入 _toolRegistry
  const toolRegistry = new Map(
    wrappedBuiltInTools.map((tool) => [tool.name, tool])
  );
  
  this._toolRegistry = toolRegistry;
  
  // 设置激活工具
  this.setActiveToolsByName([...new Set(nextActiveToolNames)]);
  // 内部：this.agent.setTools([...toolRegistry.values()])
}
```

#### ① 扩展文件调用 registerTool

```typescript

export default function(pi: ExtensionAPI) {
  pi.registerTool({
    name: 'my_custom_tool',
    label: 'My Custom Tool',
    description: '...',
    parameters: Type.Object({ ... }),
    execute: async (...) => { ... },
  });
}
```

#### ② registerTool 内部实现

```typescript
// 文件：extensions/loader.ts 第217-224行

registerTool(tool: ToolDefinition): void {
  runtime.assertActive();  // 检查扩展是否有效
  
  // ★ 核心：写入 extension.tools Map
  extension.tools.set(tool.name, {
    definition: tool,
    sourceInfo: extension.sourceInfo,
  });
  
  runtime.refreshTools();  // 刷新工具列表
}
```

#### ③ ExtensionRunner 收集所有工具

```typescript
// 文件：extensions/runner.ts 第373-384行

getAllRegisteredTools(): RegisteredTool[] {
  const toolsByName = new Map<string, RegisteredTool>();
  
  // 遍历所有扩展（包括内置扩展）
  for (const ext of this.extensions) {
    for (const tool of ext.tools.values()) {
      // 第一个注册的生效（同名工具先注册的赢）
      if (!toolsByName.has(tool.definition.name)) {
        toolsByName.set(tool.definition.name, tool);
      }
    }
  }
  
  return Array.from(toolsByName.values());
}
```

## 🔧 关键包装函数：wrapRegisteredTools

### 源码位置

| 文件 | 行号 | 函数 |
|------|------|------|
| `extensions/index.ts` | 导入 | `wrapRegisteredTools()` |
| `tools/tool-definition-wrapper.ts` | 5-19 | `wrapToolDefinition()` |

### 代码解析

```typescript
// 文件：tools/tool-definition-wrapper.ts 第5-19行

export function wrapToolDefinition<TDetails>(
  definition: ToolDefinition<any, TDetails>,
  ctxFactory?: () => ExtensionContext,
): AgentTool<any, TDetails> {
  return {
    name: definition.name,
    label: definition.label,
    description: definition.description,
    parameters: definition.parameters,
    prepareArguments: definition.prepareArguments,
    executionMode: definition.executionMode,
    
    // ★ 核心：execute 方法包装
    execute: (toolCallId, params, signal, onUpdate) =>
      definition.execute(
        toolCallId, params, signal, onUpdate, 
        ctxFactory?.() as ExtensionContext  // 注入 ExtensionContext
      ),
  };
}
```

**作用**：将 `ToolDefinition` 转换成 `AgentTool`，供核心运行时使用。

## 🗂️ 关键数据结构

### ToolDefinitionEntry

```typescript
// 文件：agent-session.ts 第235-238行

interface ToolDefinitionEntry {
  definition: ToolDefinition;    // 工具定义
  sourceInfo: SourceInfo;        // 来源信息
}
```

### RegisteredTool（扩展层）

```typescript
// 文件：extensions/types.ts 第1385-1388行

interface RegisteredTool {
  definition: ToolDefinition;    // 工具定义
  sourceInfo: SourceInfo;        // 来源信息
}
```

### SourceInfo

```typescript
// 来源标记，用于区分工具来源
{
  source: "builtin" | "extension" | "sdk" | "local" | "npm:" | "git:",
  scope: "user" | "project" | "temporary",
  path?: string,      // 扩展文件路径
  baseDir?: string,   // 基础目录
}
```

## 📋 两条路径对比表

| 维度 | 内置工具 | 扩展工具 |
|------|----------|----------|
| **创建方式** | `createXxxToolDefinition()` 工厂函数 | 用户自己写 `ToolDefinition` |
| **注册时机** | `_buildRuntime()` 时自动 | 扩展加载时调用 `pi.registerTool()` |
| **存储位置** | `_baseToolDefinitions` Map | `extension.tools` Map |
| **sourceInfo** | `{ source: "builtin" }` | `{ source: "extension", path: "..." }` |
| **包装时机** | `_refreshToolRegistry()` 时 | 同样在 `_refreshToolRegistry()` 时 |
| **最终存放** | `_toolRegistry` Map | 同样是 `_toolRegistry` Map |
| **给 Agent用** | `agent.setTools()` | 同样被设置 |
