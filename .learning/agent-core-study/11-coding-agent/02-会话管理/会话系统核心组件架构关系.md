# coding-agent 会话系统核心组件架构关系

## 概述

coding-agent 的会话系统由四个核心组件构成，形成清晰的分层架构。每个组件都有明确的职责边界和协作关系。

```
完整架构层次：

应用层 (Interactive Mode / RPC Mode / Print Mode)
    ↓ 使用
运行时管理层 (AgentSessionRuntime)
    ↓ 持有
业务操作层 (AgentSession)
    ↓ 拥有
持久化层 (SessionManager)
    ↓ 依赖
基础设施层 (AgentSessionServices)
```

---

## 一、核心组件概览

| 组件 | 层级 | 职责 | 生命周期 | 核心能力 |
|------|------|------|----------|----------|
| **AgentSessionRuntime** | 运行时管理层 | 会话生命周期管理 | 跨多个会话 | switch/new/fork/import/dispose |
| **AgentSessionServices** | 基础设施层 | 可复用的运行时服务 | cwd 绑定 | auth/settings/modelRegistry/resourceLoader |
| **AgentSession** | 业务操作层 | 单个会话的交互操作 | 单个会话 | prompt/cycleModel/compact/executeBash |
| **SessionManager** | 持久化层 | 会话历史存储与分支管理 | 单个会话 | append/buildContext/branch/createBranched |

---

## 二、各组件详解

### 2.1 AgentSessionRuntime - 运行时管理层

#### 定位
**会话管理器**，负责协调多个会话的生命周期，处理会话切换、创建、分叉等高层操作。

#### 核心职责

```typescript
class AgentSessionRuntime {
  // 持有的资源
  private _session: AgentSession;           // 当前活跃的会话
  private _services: AgentSessionServices;  // 当前活跃的服务
  private createRuntime: Factory;           // 会话创建工厂（复用）

  // 核心能力
  async switchSession(sessionPath)      // 切换到已有会话
  async newSession(options)             // 创建全新会话
  async fork(entryId, position)         // 从历史某个点分叉
  async importFromJsonl(inputPath)      // 导入会话文件
  async dispose()                       // 销毁运行时

  // 辅助方法
  setRebindSession(callback)            // 设置会话重新绑定回调
  setBeforeSessionInvalidate(callback)  // 设置会话销毁前回调
}
```

#### 内部流程（switchSession 为例）

```typescript
async switchSession(sessionPath) {
  // 1. 触发扩展事件（允许取消）
  const beforeResult = await emitBeforeSwitch("resume", sessionPath);
  if (beforeResult.cancelled) return { cancelled: true };

  // 2. 打开目标会话管理器
  const sessionManager = SessionManager.open(sessionPath);

  // 3. 验证 cwd 存在
  assertSessionCwdExists(sessionManager);

  // 4. 销毁当前会话
  await teardownCurrent("resume", sessionManager.getSessionFile());
    → emitSessionShutdownEvent("resume")
    → session.dispose()

  // 5. 创建新运行时（通过工厂）
  const result = await createRuntime({
    cwd: sessionManager.getCwd(),
    agentDir: services.agentDir,
    sessionManager,
    sessionStartEvent: { type: "session_start", reason: "resume" }
  });

  // 6. 应用新会话
  this._session = result.session;
  this._services = result.services;

  // 7. 通知 UI 重新绑定
  await rebindSession(this.session);

  return { cancelled: false };
}
```

#### 关键特点

| 特性 | 说明 |
|------|------|
| **跨会话** | 管理多个会话的生命周期，不绑定到特定会话 |
| **事件驱动** | 触发 session_before_switch、session_shutdown 等扩展事件 |
| **工厂模式** | 保存 createRuntime 工厂，在切换时复用创建逻辑 |
| **动态绑定** | _session 和 _services 会动态变化 |
| **高层协调** | 协调服务创建、会话创建、扩展通知、UI 绑定 |

#### 使用场景

```typescript
// 1. 创建运行时
const runtime = await createAgentSessionRuntime(createRuntime, {
  cwd: "/project",
  agentDir: "/home/.pi/agent",
  sessionManager
});

// 2. 切换会话
await runtime.switchSession("/path/to/old-session.jsonl");

// 3. 创建新会话
await runtime.newSession({ parentSession: currentSessionFile });

// 4. 从历史分叉
await runtime.fork(entryId, { position: "before" });

// 5. 导入会话文件
await runtime.importFromJsonl("/path/to/exported.jsonl");

// 6. 清理
await runtime.dispose();

// 7. 访问当前会话（通过 runtime.session）
const session = runtime.session;
await session.prompt("hello");
```

### 2.2 AgentSessionServices - 基础设施层

#### 定位
**服务工厂**，创建与 cwd 绑定的可复用运行时服务，提供会话运行所需的基础设施。

#### 核心职责

```typescript
interface AgentSessionServices {
  cwd: string;                          // 工作目录
  agentDir: string;                     // Agent 配置目录
  authStorage: AuthStorage;             // 认证存储（API keys）
  settingsManager: SettingsManager;     // 设置管理器
  modelRegistry: ModelRegistry;         // 模型注册表
  resourceLoader: ResourceLoader;       // 资源加载器（扩展、MCP）
  diagnostics: Diagnostic[];            // 创建过程中的诊断信息
}

async function createAgentSessionServices(options) {
  // 创建流程
  const authStorage = AuthStorage.create(agentDir + "auth.json");
  const settingsManager = SettingsManager.create(cwd, agentDir);
  const modelRegistry = ModelRegistry.create(authStorage, agentDir + "models.json");
  const resourceLoader = new DefaultResourceLoader({ cwd, agentDir, settingsManager });

  await resourceLoader.reload();  // 加载扩展、MCP服务器

  // 注册扩展提供的模型提供者
  for (const { name, config } of extensions.pendingProviderRegistrations) {
    modelRegistry.registerProvider(name, config);
  }

  return { cwd, agentDir, authStorage, settingsManager, modelRegistry, resourceLoader };
}
```

#### 关键特点

| 特性 | 说明 |
|------|------|
| **cwd 绑定** | 与特定工作目录关联，cwd 改变需重建 |
| **会话无关** | 不包含会话特定配置（模型、工具等） |
| **可复用** | 一次创建，可在多个会话间共享 |
| **高成本** | 涉及文件读取、扩展加载等重操作 |
| **基础设施** | 提供"环境"而非"应用" |

#### 使用场景

```typescript
// 1. 创建服务（一次）
const services = await createAgentSessionServices({
  cwd: "/project/path",
  extensionFlagValues: new Map([["debug", true]])
});

// 2. 检查诊断信息
if (services.diagnostics.some(d => d.type === "error")) {
  console.error("服务创建失败");
  return;
}

// 3. 复用服务创建多个会话
const session1 = await createAgentSessionFromServices({
  services,
  model: devModel,
  tools: ["read", "bash"]
});

const session2 = await createAgentSessionFromServices({
  services,
  model: testModel,
  tools: ["read"]
});

// 4. cwd 改变时重建服务
const newServices = await createAgentSessionServices({ cwd: newCwd });
```

### 2.3 AgentSession - 业务操作层

#### 定位
**会话实例**，单个会话的主接口，封装所有业务操作：对话、工具、压缩、模型切换等。

#### 核心职责

```typescript
class AgentSession {
  // 拥有的资源
  readonly agent: Agent;                 // AI 模型交互引擎
  readonly sessionManager: SessionManager; // 持久化管理器
  readonly settingsManager: SettingsManager;

  // 核心能力分类

  // 1. 核心交互
  async prompt(text, options)           // 发送消息给 Agent
  async abort()                          // 中断当前执行

  // 2. 会话管理
  async switchSession(sessionId)        // 切换会话
  async branch(options)                 // 创建分支
  setSessionName(name)                  // 设置会话名称
  getSessionStats()                     // 获取会话统计

  // 3. 工具管理
  getActiveToolNames()                  // 获取当前启用的工具
  getAllTools()                         // 获取所有可用工具
  setActiveToolsByName(toolNames)       // 设置启用的工具

  // 4. 模型管理
  async cycleModel(direction)           // 切换模型
  getThinkingLevel()                    // 获取思考级别
  setThinkingLevel(level)               // 设置思考级别

  // 5. 压缩管理
  async compact(instructions)           // 手动压缩
  abortCompaction()                     // 中断压缩
  setAutoCompactionEnabled(enabled)     // 启用/禁用自动压缩
  getContextUsage()                     // 获取上下文使用情况

  // 6. 消息队列
  queueSteeringMessage(text)            // 添加中断消息
  queueFollowUpMessage(text)            // 添加后续消息
  clearQueue()                          // 清空队列

  // 7. Bash 执行
  async executeBash(command, onChunk)   // 执行 Bash 命令
  abortBash()                           // 中断 Bash

  // 8. 状态访问
  get state                             // Agent 状态
  get cwd                               // 工作目录
  get messages                          // 消息历史
  getLastAssistantText()                // 最后的助手消息

  // 9. 事件订阅
  subscribe(listener)                   // 订阅会话事件

  // 10. 扩展系统
  async reloadExtensions()              // 重载扩展
  hasExtensionHandlers(eventType)       // 检查扩展处理器
}
```

#### 关键特点

| 特性 | 说明 |
|------|------|
| **单会话绑定** | 只管理当前这一个会话 |
| **业务接口** | 你所有操作都通过它 |
| **事件驱动** | 发出 message_start、tool_execution_start 等事件 |
| **状态管理** | 维护 Agent 的运行状态 |
| **组合模式** | 拥有 Agent 和 SessionManager |

#### 内部协作（prompt 为例）

```typescript
async prompt(text, options) {
  // 1. 解析消息
  const userMessage = parseUserMessage(text);

  // 2. 发送给 Agent（AI 引擎）
  await this.agent.process(userMessage);

  // 3. Agent 内部流程
  agent.process()
    → streamFn(model, context)  // 调用模型 API
    → receive response          // 接收响应
    → execute tools             // 执行工具调用

  // 4. SessionManager 持久化
  sessionManager.appendMessage(userMessage);
  sessionManager.appendMessage(assistantMessage);

  // 5. 发出事件
  emit({ type: "message_start", message });
  emit({ type: "message_update", content });
  emit({ type: "message_end" });
}
```

#### 使用场景

```typescript
// 通过 AgentSession 进行所有业务操作

// 1. 发送消息
await session.prompt("帮我写一个函数");

// 2. 切换模型
await session.cycleModel();

// 3. 执行命令
const result = await session.executeBash("npm test");

// 4. 压缩上下文
if (session.getContextUsage().percentage > 80) {
  await session.compact("保留最近的代码修改");
}

// 5. 订阅事件
session.subscribe((event) => {
  if (event.type === "message_update") {
    console.log(event.content);
  }
});

// 6. 创建分支
await session.branch({ name: "alternative-approach" });
```

### 2.4 SessionManager - 持久化层

#### 定位
**会话历史管理器**，负责会话持久化、历史存储、分支管理、上下文构建。

#### 核心职责

```typescript
class SessionManager {
  // 持久化状态
  private sessionId: string;
  private sessionFile: string;          // 会话文件路径
  private fileEntries: FileEntry[];     // 所有历史条目

  // 树结构状态（支持分支）
  private byId: Map<string, SessionEntry>;  // ID → 条目映射
  private leafId: string | null;            // 当前叶子节点

  // 核心能力分类

  // 1. 基本信息访问
  getSessionId()                        // 获取会话 ID
  getSessionName()                      // 获取会话名称
  getSessionFile()                      // 获取会话文件路径
  getCwd()                              // 获取工作目录

  // 2. 历史追加
  appendMessage(message)                // 追加消息到历史
  appendModelChange(provider, modelId)  // 记录模型切换
  appendThinkingLevelChange(level)      // 记录思考级别变化
  appendCompaction(...)                 // 记录压缩操作
  appendCustomEntry(customType, data)   // 扩展自定义数据
  appendSessionInfo(name)               // 会话元信息

  // 3. 历史查询
  getEntries()                          // 获取所有条目
  getEntry(id)                          // 获取单个条目
  getBranch(fromId)                     // 获取当前分支路径
  getChildren(parentId)                 // 获取子节点
  getLeafId()                           // 获取当前叶子 ID

  // 4. 上下文构建
  buildSessionContext()                 // 构建发送给 LLM 的上下文
  // {
  //   messages: [...],  // 发送给模型的消息
  //   model: {...},     // 当前模型信息
  //   thinkingLevel: "medium"
  // }

  // 5. 分支管理
  branch(newLeafId)                     // 切换到新分支
  branchWithSummary(...)                // 带摘要的分支
  createBranchedSession(leafId)         // 创建分支会话文件
  resetLeaf()                           // 重置叶子节点

  // 6. 会话管理
  newSession(options)                   // 创建新会话
  setSessionFile(sessionFile)           // 切换到已有会话文件

  // 7. 树结构查询
  getTree()                             // 获取完整树结构
  getLabel(id)                          // 获取标签
  appendLabelChange(targetId, label)    // 设置标签
}
```

#### 条目类型

```typescript
type SessionEntry =
  | SessionMessageEntry         // 消息条目
  | ThinkingLevelChangeEntry    // 思考级别变化
  | ModelChangeEntry            // 模型切换
  | CompactionEntry             // 压缩摘要
  | BranchSummaryEntry          // 分支摘要
  | CustomEntry                 // 扩展自定义数据
  | CustomMessageEntry          // 扩展注入的消息（进入 LLM 上下文）
  | LabelEntry                  // 用户标签/书签
  | SessionInfoEntry            // 会话元信息（名称等）
```

#### 关键特点

| 特性 | 说明 |
|------|------|
| **持久化** | 所有操作都写入 JSONL 文件 |
| **树结构** | 通过 id/parentId 维护分支树 |
| **历史重建** | 从条目构建发送给 LLM 的上下文 |
| **分支支持** | 支持从任意历史点创建分支 |
| **扩展友好** | CustomEntry 支持扩展存储自定义数据 |

#### 内部流程（buildSessionContext）

```typescript
buildSessionContext() {
  // 1. 获取当前分支的所有条目
  const branchEntries = this.getBranch(this.leafId);

  // 2. 构建消息列表
  const messages = [];

  for (const entry of branchEntries) {
    if (entry.type === "message") {
      // 直接添加消息
      messages.push(entry.message);
    } else if (entry.type === "compaction") {
      // 添加压缩摘要（作为用户消息）
      messages.push(createCompactionSummaryMessage(entry.summary));
    } else if (entry.type === "branch_summary") {
      // 添加分支摘要
      messages.push(createBranchSummaryMessage(entry.summary));
    } else if (entry.type === "model_change") {
      // 记录模型变化（用于恢复）
      currentModel = { provider: entry.provider, modelId: entry.modelId };
    } else if (entry.type === "thinking_level_change") {
      // 记录思考级别变化
      currentThinkingLevel = entry.thinkingLevel;
    } else if (entry.type === "custom_message") {
      // 扩展注入的消息
      messages.push(createCustomMessage(entry.content));
    }
  }

  return {
    messages,
    model: currentModel,
    thinkingLevel: currentThinkingLevel
  };
}
```

#### 使用场景

```typescript
// SessionManager 主要被 AgentSession 内部调用
// 但也可以直接访问（通过 session.sessionManager）

const manager = session.sessionManager;

// 1. 获取基本信息
const sessionId = manager.getSessionId();
const sessionName = manager.getSessionName();

// 2. 获取历史
const entries = manager.getEntries();

// 3. 获取当前分支
const branch = manager.getBranch();

// 4. 查询树结构
const tree = manager.getTree();
const children = manager.getChildren(parentId);

// 5. 创建分支
manager.branchWithSummary(branchFromId, "探索新方案");

// 6. 构建上下文（发送给 LLM）
const context = manager.buildSessionContext();
```

## 三、组件之间的关系

### 3.1 组合关系

```
AgentSessionRuntime
    ├─ 拥有 _session: AgentSession (动态绑定)
    ├─ 拥有 _services: AgentSessionServices (动态绑定)
    ├─ 保存 createRuntime: Factory (复用)
    └─ 管理 生命周期回调

AgentSession
    ├─ 拥有 agent: Agent (AI 引擎)
    ├─ 拥有 sessionManager: SessionManager (持久化)
    ├─ 拥有 settingsManager: SettingsManager (配置)
    └─ 拥有 resourceLoader: ResourceLoader (资源)

SessionManager
    ├─ 持有 sessionFile: string (持久化文件)
    ├─ 持有 fileEntries: Entry[] (历史条目)
    └─ 持有 byId: Map<string, Entry> (树结构)

AgentSessionServices
    ├─ 拥有 authStorage: AuthStorage
    ├─ 拥有 settingsManager: SettingsManager
    ├─ 拥有 modelRegistry: ModelRegistry
    └─ 拥有 resourceLoader: ResourceLoader
```

### 3.2 依赖关系

```
AgentSessionRuntime
    ↓ 依赖
createAgentSessionServices (创建服务)
    ↓ 依赖
createAgentSessionFromServices (创建会话)
    ↓ 依赖
AgentSession (会话实例)
    ↓ 依赖
SessionManager (持久化)
```

### 3.3 数据流

#### 创建流程

```
1. createAgentSessionServices({ cwd })
   ↓ 返回
   AgentSessionServices

2. createAgentSessionFromServices({ services, model, tools })
   ↓ 调用
   createAgentSession({ ... })

3. createAgentSession 内部
   ├─ 创建 Agent (AI 引擎)
   ├─ 恢复历史 (从 SessionManager)
   ├─ 初始化工具
   └↓ 返回
   AgentSession

4. AgentSession 内部
   ├─ 拥有 SessionManager
   ├─ 拥有 Agent
   └↓ 封装
   业务操作接口

5. AgentSessionRuntime 包装
   ├─ 持有 AgentSession
   ├─ 持有 AgentSessionServices
   ├─ 保存工厂
   └↓ 提供
   生命周期管理
```

#### 运行流程（发送消息）

```
用户调用
    ↓
session.prompt("hello")
    ↓
AgentSession 内部
    ├─ 解析消息
    ├─ 发送给 Agent
    ├↓ Agent 处理
        ├─ 调用模型 API (streamFn)
        ├─ 接收响应
        ├─ 执行工具调用
        ├↓ 发出事件
            message_start
            message_update
            message_end
    ↓
SessionManager 持久化
    ├─ appendMessage(userMessage)
    ├─ appendMessage(assistantMessage)
    ├↓ 写入文件
        session.jsonl (追加)
    ↓
返回结果给用户
```

#### 切换流程（runtime.switchSession）

```
用户调用
    ↓
runtime.switchSession("/path/to/old.jsonl")
    ↓
AgentSessionRuntime 内部
    ├─ emitBeforeSwitch("resume") // 扩展事件
    ├─ SessionManager.open(sessionPath) // 打开目标
    ├↓ teardownCurrent("resume")
        ├─ emitSessionShutdownEvent("resume")
        ├─ session.dispose()
    ├↓ createRuntime({ cwd, sessionManager })
        ├↓ createAgentSessionServices({ cwd })
            ├─ 加载配置
            ├─ 加载扩展
            ├↓ 返回 services
        ├↓ createAgentSessionFromServices({ services })
            ├↓ 创建 AgentSession
            ├↓ 返回 session
    ├↓ apply(result)
        ├─ _session = result.session
        ├─ _services = result.services
    ├↓ rebindSession(session)
        ├─ 通知 UI 重新绑定
    ↓
返回 { cancelled: false }
```

## 四、职责边界

### 4.1 谁负责什么？

| 操作 | 负责组件 | 原因 |
|------|----------|------|
| **切换会话** | AgentSessionRuntime | 需要协调销毁、创建、事件、UI |
| **发送消息** | AgentSession | 业务操作，单会话范围 |
| **持久化消息** | SessionManager | 持久化职责 |
| **创建服务** | createAgentSessionServices | 基础设施初始化 |
| **创建会话** | createAgentSessionFromServices | 会话实例创建 |
| **切换模型** | AgentSession | 业务操作，但 SessionManager 记录 |
| **创建分支** | AgentSessionRuntime | 需要创建新会话实例 |
| **压缩上下文** | AgentSession | 业务操作，但 SessionManager 记录摘要 |
| **导入会话** | AgentSessionRuntime | 生命周期管理 |

### 4.2 不应该做什么？

| 组件 | 不应该做的 | 原因 |
|------|-----------|------|
| **AgentSession** | 切换到其他会话文件 | 它只管理当前会话，切换由 Runtime 负责 |
| **SessionManager** | 发送消息给 Agent | 它只负责存储，不负责业务操作 |
| **AgentSessionServices** | 创建 Agent 实例 | 它只提供基础设施，不创建业务实例 |
| **AgentSessionRuntime** | 执行具体的工具调用 | 它管理生命周期，不处理业务细节 |

## 五、完整使用示例

### 5.1 标准创建流程

```typescript
// 1. 创建服务工厂函数
const createRuntime: CreateAgentSessionRuntimeFactory = async (options) => {
  // 1.1 创建服务
  const services = await createAgentSessionServices({
    cwd: options.cwd,
    agentDir: options.agentDir,
    extensionFlagValues: flagValues
  });

  // 1.2 检查诊断
  if (services.diagnostics.some(d => d.type === "error")) {
    throw new Error("服务创建失败");
  }

  // 1.3 创建会话
  const result = await createAgentSessionFromServices({
    services,
    sessionManager: options.sessionManager,
    model: services.modelRegistry.find("anthropic", "claude-sonnet-4"),
    thinkingLevel: "high",
    tools: ["read", "bash", "edit", "write"],
    sessionStartEvent: options.sessionStartEvent
  });

  // 1.4 返回运行时结果
  return {
    session: result.session,
    services: result.services,
    diagnostics: services.diagnostics,
    extensionsResult: result.extensionsResult,
    modelFallbackMessage: result.modelFallbackMessage
  };
};

// 2. 创建会话管理器
const sessionManager = SessionManager.create(cwd, sessionDir);

// 3. 创建运行时
const runtime = await createAgentSessionRuntime(createRuntime, {
  cwd: "/project/path",
  agentDir: "/home/.pi/agent",
  sessionManager
});

// 4. 访问当前会话
const session = runtime.session;

// 5. 业务操作
await session.prompt("帮我分析项目架构");
await session.cycleModel();
const result = await session.executeBash("npm test");

// 6. 会话管理
await runtime.newSession();
await runtime.switchSession("/path/to/old.jsonl");
await runtime.fork(entryId);

// 7. 清理
await runtime.dispose();
```

## 六、架构设计理念

### 6.1 分层优势

1. **职责清晰**
   - Runtime：会话生命周期
   - Session：业务操作
   - SessionManager：持久化
   - Services：基础设施

2. **复用性强**
   - Services 可跨会话共享
   - Runtime 工厂可复用
   - SessionManager 可独立使用

3. **扩展性好**
   - 新增运行模式只需实现 Runtime
   - 新增业务能力只需扩展 Session
   - 新增持久化格式只需修改 SessionManager

4. **测试友好**
   - 各层可独立测试
   - Mock 某层不影响其他层

### 6.2 关键设计决策

#### 为什么需要 Runtime？

- 会话切换涉及销毁、创建、事件、UI 绑定，需要高层协调
- AgentSession 只管理单个会话，不应该处理切换逻辑
- Runtime 提供统一的会话生命周期管理

#### 为什么 Services 和 Session 分离？

- Services 创建成本高（读文件、加载扩展）
- Session 创建成本低（主要是配置组装）
- Services 可复用，Session 每次不同
- cwd 改变只需重建 Services

#### 为什么 SessionManager 独立？

- 持久化逻辑复杂（树结构、迁移、压缩）
- 与业务逻辑解耦
- 可独立演进（新增条目类型、优化存储）

## 八、总结

### 核心要点

1. **四层架构**
   - Runtime：会话管理（跨会话）
   - Session：业务操作（单会话）
   - SessionManager：持久化（存储）
   - Services：基础设施（环境）

2. **组合关系**
   - Runtime 拥有 Session 和 Services（动态）
   - Session 拥有 Agent 和 SessionManager（固定）

3. **职责边界**
   - Runtime：会话生命周期
   - Session：业务交互
   - SessionManager：历史存储
   - Services：环境提供

4. **数据流**
   - 创建：Services → Session → Runtime
   - 运行：Session → Agent + SessionManager
   - 切换：Runtime → 销毁旧 + 创建新

### 使用原则

- **切换会话** → 用 Runtime
- **发送消息** → 用 Session
- **查询历史** → 用 SessionManager（通过 Session）
- **创建环境** → 用 Services
- **管理生命周期** → 用 Runtime

### 架构精髓

```
AgentSessionRuntime  = 生命周期管理器（管理多个会话）
AgentSession         = 业务操作接口（单个会话）
SessionManager       = 持久化引擎（历史存储）
AgentSessionServices = 基础设施提供者（环境）
```

这四者协同工作，实现了灵活、可扩展、易维护的会话系统架构。
